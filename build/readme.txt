===============================
Описание файлов сборки проектов

Список файлов:
1. master.build - основной файл сборки, содержит цели и задачи сборки проектов и очистки от временных файлов;
2. version.build - содержит задачи для для получения и обновления всех файлов содержащих версию;
3. properties.build - содержит только пути к проектам и названия файлов используемые при сборке;
4. version.targets - импортируется в те файлы, где нужна строка с версией;
4. version.vcs.targets - сгенерированный файл, содержит часть версии из Git;
5. version.vcs.template.targets - шаблон для генерации version.build.targets.

===============================
По оформлению

Все пути в файлах *.build должны оканчиваться "\"

===============================
По задачам

Любую сборку проекта или инсталлятора следует начинать с вызова build/master.build /t:GetVersionFromGit и build/master.build /t:BuildVersionFiles.
После выполнения скрипиов генерируется файл (actualversion.txt) с строковым представлением версий в корневой папке проекта.
